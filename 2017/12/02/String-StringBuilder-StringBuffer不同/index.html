String,StringBuilder,StringBuffer不同

发表于 2017-12-02 更新于 2020-04-03 分类于 java 阅读次数：
本文字数： 987 阅读时长 ≈ 1 分钟

最近简单浏览了String的源码,很清楚地知道为何String是不可变的,同时也简单浏览了和String相关比较密切的StringBuilder和StringBuffer的代码,在此记简单记录以下三者的区别.

String

在JDK源码中String 定义为final类型,其内部主要维护了 final char[ ] ,由于类型设置为final因此String是不可变的,每次重新为String引用赋值.而String内的大多数方法都是对char[]进行的操作.

StringBuilder

StringBuilder代表字符串变量其内部维护的也是一个char[] ,只是没有被final修饰,是可以改变的.下面是其类图

StringBuffer

StringBuffer与StringBuilder从源码的角度及其相似,都是实现AbstractStringBuilder,提供了与StringBuilder相同的方法,但是其所有的非构造方法都有synchronized关键字.
